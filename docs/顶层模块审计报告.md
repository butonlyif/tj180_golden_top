# TJ180 Golden Top — 顶层模块审计报告

| 项目 | 内容 |
|------|------|
| **审计日期** | 2026-07-18 |
| **审计问题** | 现在有两个 top 层吗？ |
| **结论** | ❌ **没有**。工程只有 **一个** 顶层模块：`tj180_golden_top` |

---

## 1. 工程顶层配置（权威来源）

`tj180_golden_top.xml`（Efinity 工程文件）中明确定义：

```xml
<efx:design_info>
    <efx:top_module name="tj180_golden_top"/>
    <efx:design_file name="tj180_golden_top.v" version="default" library="default"/>
</efx:design_info>
```

- **`<top_module>`** = `tj180_golden_top`，且唯一
- **`<design_file>`** 仅列出 `tj180_golden_top.v`，不包含 `rtl/top.sv` 或 `rtl/pll_top.sv`

## 2. 含 "top" 的源文件清单

| 文件 | 模块名 | 是否在工程 XML 中 | 是否编译 | 状态 |
|------|--------|------------------|---------|------|
| `tj180_golden_top.v` | `tj180_golden_top` | ✅ 是（design_file + top_module） | ✅ 是 | **唯一顶层** |
| `rtl/top.sv` | `top` | ❌ 否 | ❌ 否 | **已废弃**（Stage 0 占位，LED 慢闪） |
| `rtl/pll_top.sv` | `pll_top` | ❌ 否 | ❌ 否 | **已废弃**（Stage 0 占位，时钟直通） |

> `rtl/top.sv` 和 `rtl/pll_top.sv` 是 `awesom new-project` 生成的板卡模板 Stage 0 初始文件。
> Stage 1 起已由 `tj180_golden_top.v` + `rtl/clk_rst/pll_sys_wrapper.sv` 完全替代。
> 两个文件已标注 `DEPRECATED` 头注释，保留仅为历史参考。

## 2b. Stage 2 新增子模块清单

| 文件 | 模块 | 用途 |
|------|------|------|
| `rtl/cdc/apb_to_axilite.sv` | `apb_to_axilite` | APB3 Slave → AXI4-Lite Master 桥，连接 SoC APB Slave 0 与 CSI RX 寄存器 |
| `rtl/ip_wrappers/hard_csi_rx_wrapper.sv` | `hard_csi_rx_wrapper` | 包装 `hard_csi_rx` IP（v5.5.1），分组 AXI-Lite / DPHY RX / Pixel / IRQ 端口 |

## 2c. Stage 3 新增子模块清单

| 文件 | 模块 | 用途 |
|------|------|------|
| `rtl/cdc/async_fifo.sv` | `async_fifo` | 双时钟异步 FIFO（DW=64 / 深度 2048 / sideband 8-bit），RX pixel 域 → TX pixel 域 |
| `rtl/ctrl/loopback_ctrl.sv` | `loopback_ctrl` | RX→FIFO→TX loopback 控制器；两时钟域设计，重建 TX line/haddr/frame + vsync/hsync |
| `rtl/ip_wrappers/hard_csi_tx_wrapper.sv` | `hard_csi_tx_wrapper` | 包装 `hard_csi_tx` IP（v5.14, 2-lane），分组 AXI-Lite / Pixel / DPHY TX / IRQ 端口 |

## 3. 模块层级树（编译参与部分）

```
tj180_golden_top (tj180_golden_top.v)          ← 唯一顶层
├── pll_sys_wrapper (rtl/clk_rst/pll_sys_wrapper.sv)
├── pll_ddr_wrapper (rtl/clk_rst/pll_ddr_wrapper.sv)
├── rst_sync × 5 (rtl/clk_rst/rst_sync.sv)
│     ├── u_rst_sys       (sys_clk 域)
│     ├── u_rst_ddr       (ddr_clk 域)
│     ├── u_rst_byte_hs   (clk_byte_hs 域, Stage 2)
│     ├── u_rst_pixel     (clk_pixel_rx 域, Stage 2)
│     └── u_rst_pixel_tx  (clk_pixel_tx 域, Stage 3 新增)
├── sapphire_soc_wrapper (rtl/ip_wrappers/sapphire_soc_wrapper.sv)
│     └── soc (ip/sapphire_soc/soc.v)
├── ddr_ctrl_wrapper (rtl/ip_wrappers/ddr_ctrl_wrapper.sv)
├── apb_to_axilite (rtl/cdc/apb_to_axilite.sv)             ← Stage 2
│     └── u_apb_to_axilite_csi (SoC APB Slave 0 → CSI RX AXI-Lite)
├── hard_csi_rx_wrapper (rtl/ip_wrappers/hard_csi_rx_wrapper.sv)  ← Stage 2
│     └── u_hard_csi_rx
│           └── hard_csi_rx (ip/hard_csi_rx/hard_csi_rx.sv)
│                 └── efx_csi2_rx_top (IP v5.5.1 加密实例)
├── async_fifo (rtl/cdc/async_fifo.sv)                     ← Stage 3 新增
│     └── u_async_pixel_fifo (RX pixel → TX pixel CDC, 64-bit + 8-bit sideband)
├── loopback_ctrl (rtl/ctrl/loopback_ctrl.sv)              ← Stage 3 新增
│     └── u_loopback_ctrl (RX 写 FIFO / TX 读 FIFO + 计数重建)
└── hard_csi_tx_wrapper (rtl/ip_wrappers/hard_csi_tx_wrapper.sv) ← Stage 3 新增
      └── u_hard_csi_tx
            └── hard_csi_tx (ip/hard_csi_tx/hard_csi_tx.sv)
                  └── efx_csi2_tx_top (IP v5.14 加密实例, 2-lane)
```

> `rtl/top.sv` (`module top`) 和 `rtl/pll_top.sv` (`module pll_top`) **不在**此树中。

## 4. 采取措施

1. `rtl/top.sv` — 添加 `⚠️ DEPRECATED` 头注释，明确指向 `tj180_golden_top.v`
2. `rtl/pll_top.sv` — 添加 `⚠️ DEPRECATED` 头注释，明确指向 `pll_sys_wrapper.sv`
3. 未修改 `tj180_golden_top.xml`（无需变动，本来就没有引用这两个文件）
4. 本报告作为审计记录存档

```bash
# rtl/pll_top.sv:module pll_top (  ← 已废弃，未编译
# tj180_golden_top.v:module tj180_golden_top  ← 唯一顶层
```
